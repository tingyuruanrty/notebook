IDE trick
the most basic use of git(local repository) and github(remote repository) three steps: 1 add equals to add button in the branching interface of code space what actually did is add the file to the track list, tell the git that we want to track the change of this file, the reason is there might have multiple file in a directory, we might only want to track some of them code: git add filename.suffix add won't affect the content of repository 2 commit the first big blue button in the branching interface of code space commit would actually put the file in the repository(not exactly, think it as took photos and record difference between photos) with a message we assigned, the message is mandatory, we can't commit without message. the message is really important, because it would be the only thing that make sense in the log code: git commit -m "message" commit would save the changes and store it in the local repository 3 push (pull(they have same syntax, the pull is update the local repo to remote repo, which is converse to push)) be eclipsed beside the big blue button in the branching interface of code space if the branch doesn't exist in remote repo, it would create it in remote repo, the default repo name is origin, the name for main branch is main notice: we need to have the branch on local repo code: git push -u reponame branchname "-u" is optional, with -u, we don't need to specify the reponame and branch anymore, really useful push would hand in the content of local repository to the remote repository (edited)
